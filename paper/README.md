# Статья

Статья (формат `article`) о репозитории go-lgbm-serving: зачем он, какой приём
несущий (линковка той же `lib_lightgbm` из venv Python), что измерено. Это
межсистемный proof-of-concept (Python обучает -> Go подаёт -> клиенты зовут), и
статья - его развёрнутое изложение.

## Сборка

```sh
cd paper
latexmk -pdf main.tex     # pdflatex + biber; кириллица через T2A + babel
```

Нужны TeX-дистрибутив с поддержкой кириллицы (T2A, babel-russian) и `biber`.

## Структура

- `main.tex` - преамбула и сборка секций.
- `sections/` - текст по разделам: цель -> данные -> метод -> альтернативы ->
  результат -> конвейер -> охват -> заключение (зеркалит docs/DESIGN.md).
- `assets/refs.bib` - библиография.
- `figures/` - **генерируются ноутбуками** (`training/notebooks/`). Чтобы числа и графики
  статьи не расходились с harness, ноутбуки должны сохранять фигуры (ROC/PR,
  beeswarm, latency) сюда через `savefig`. Коммитить ли готовые PDF/PNG - решается
  отдельно (ноутбуки тяжёлые, требуют большой машины).

## Решение: статья поглощает docs/DESIGN.md

Принято: единственный длинный источник рационала - эта статья. `docs/DESIGN.md`
пока остаётся живым (статья ещё не написана). **В фазе написания**: перенести
рационал из `docs/DESIGN.md` в `sections/`, затем заменить `docs/DESIGN.md`
коротким указателем на `paper/`. До этого момента DESIGN.md - источник.

## Как писать

Прозу - через skill `writer` (thesis-first, канон цитирования). Русскую
типографику (неразрывные пробелы, ёлочки, тире, cleveref) - через skill
`ru-latex`. Каркас секций уже содержит TODO с указанием, откуда брать содержимое.
